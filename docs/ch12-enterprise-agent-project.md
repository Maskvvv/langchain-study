# 第十二章：企业知识库 Agent 项目

最后一章建议做一个完整项目：企业知识库 Agent。它覆盖 Agent 岗位常见能力，也适合作为作品集。

## 12.1 项目目标

构建一个能回答企业内部知识、调用业务工具、支持人工审批、可观测和可评测的 Agent。

用户可以问：

- “公司的报销流程是什么？”
- “帮我查一下订单 A123 的状态。”
- “根据这份会议纪要生成待办并发给项目群。”

Agent 应该能判断：

- 什么时候只需要 RAG 回答
- 什么时候需要调用工具
- 什么时候必须人工审批
- 什么时候应该拒绝

## 12.2 推荐架构

```
FastAPI / Web UI
  ↓
会话管理与鉴权
  ↓
LangGraph Agent
  ├─ 意图识别节点
  ├─ RAG 检索节点
  ├─ 工具调用节点
  ├─ 人工审批节点
  ├─ 回答生成节点
  └─ 日志与评测节点
  ↓
业务系统 / 向量数据库 / LangSmith
```

## 12.3 Graph 设计

建议 State 至少包含：

| 字段 | 说明 |
|------|------|
| messages | 对话消息 |
| user_id | 当前用户 |
| intent | 意图 |
| retrieved_docs | 检索结果 |
| tool_calls | 工具调用计划 |
| approval_status | 审批状态 |
| final_answer | 最终回答 |

## 12.4 工具清单

| 工具 | 风险等级 | 说明 |
|------|----------|------|
| search_knowledge_base | 低 | 查询知识库 |
| query_order | 中 | 查询订单，需要用户权限 |
| create_task | 中 | 创建待办，需要确认 |
| send_message | 高 | 发送消息，需要审批 |
| update_record | 高 | 修改业务数据，需要审批 |

## 12.5 评测集设计

至少准备四类样例：

1. **普通知识问答**：检查 RAG 是否能答对。
2. **工具调用问题**：检查是否选对工具和参数。
3. **权限问题**：检查是否拒绝越权请求。
4. **注入攻击**：检查是否不会执行恶意指令。

## 12.6 里程碑

| 里程碑 | 目标 |
|--------|------|
| M1 | 完成基础 RAG 问答 |
| M2 | 加入工具调用 |
| M3 | 用 LangGraph 重构流程 |
| M4 | 加入 checkpoint 和人工审批 |
| M5 | 接入 LangSmith tracing |
| M6 | 建立评测集和回归测试 |
| M7 | 用 FastAPI 暴露服务 |

## 12.7 示例入口

```bash
uv run python src/ch12_enterprise_agent_project/01_architecture_walkthrough.py
```

这个示例不会实现完整系统，而是打印推荐的模块拆分和开发顺序。你可以把它当作后续实战项目的任务清单。
