# 第三章：Retrieval — 让 LLM 拥有记忆

## 3.1 为什么需要 Retrieval？

LLM 的两大局限：
1. **知识截止**：训练数据有截止日期，不知道最新信息
2. **上下文窗口有限**：无法一次性处理大量文档

RAG（Retrieval-Augmented Generation，检索增强生成）通过"先检索，再生成"解决这些问题：

```
用户提问 → [检索相关文档] → [文档+问题一起送入LLM] → 生成回答
```

## 3.2 RAG 完整流程

```
┌──────────────── 离线索引阶段 ────────────────┐
│                                               │
│  文档 → Document Loader → Text Splitter      │
│           ↓                  ↓                │
│       原始文档          文档切片(Chunks)       │
│                               ↓                │
│                    Embedding Model             │
│                               ↓                │
│                      Vector Store             │
└───────────────────────────────────────────────┘

┌──────────────── 在线查询阶段 ────────────────┐
│                                               │
│  用户问题 → Embedding → 向量相似度检索        │
│                          ↓                    │
│                     相关文档切片               │
│                          ↓                    │
│            问题 + 文档 → LLM → 回答           │
└───────────────────────────────────────────────┘
```

## 3.3 核心组件详解

### Document Loader（文档加载器）

将各种格式的数据源加载为 LangChain 的 Document 对象：

| Loader | 数据源 |
|--------|--------|
| TextLoader | 纯文本文件 |
| PyPDFLoader | PDF 文件 |
| CSVLoader | CSV 文件 |
| WebBaseLoader | 网页 |
| DirectoryLoader | 目录下所有文件 |

### Text Splitter（文本切分器）

将长文档切分为适合 LLM 处理的小块：

| Splitter | 特点 |
|----------|------|
| RecursiveCharacterTextSplitter | 递归切分，优先按段落→句子→字符（最常用） |
| CharacterTextSplitter | 按指定字符切分 |
| TokenTextSplitter | 按 token 数量切分 |

**关键参数**：
- `chunk_size`：每个块的最大大小
- `chunk_overlap`：相邻块之间的重叠大小（保持上下文连贯）

### Embedding（嵌入模型）

将文本转换为高维向量，语义相似的文本向量距离更近：

```
"猫是一种宠物" → [0.12, -0.34, 0.56, ...]  (1536维)
"狗是一种宠物" → [0.11, -0.32, 0.55, ...]  (语义相近，向量相似)
"今天股票大跌" → [0.89, 0.12, -0.67, ...]  (语义不同，向量差异大)
```

### Vector Store（向量数据库）

存储文档的嵌入向量，支持相似度检索：

| Vector Store | 特点 |
|--------------|------|
| Chroma | 开源，适合开发和小规模使用 |
| FAISS | Facebook 开源，高性能，纯内存 |
| Pinecone | 云服务，适合生产环境 |
| Milvus | 开源，适合大规模生产环境 |

## 3.4 检索策略

| 策略 | 说明 |
|------|------|
| 相似度检索 | 最基础，按向量距离排序 |
| MMR（最大边际相关性） | 兼顾相关性和多样性，避免返回内容重复的文档 |
| 多查询检索 | 用 LLM 生成多个查询，扩大检索范围 |

## 3.5 原理深入：为什么 Embedding 能表示语义？

Embedding 模型通过在海量文本上训练，学会了词语和句子之间的语义关系。核心思想是：

- **分布假设**：出现在相似上下文中的词语具有相似的含义
- **向量空间**：语义关系可以用向量空间中的几何关系表示
  - king - man + woman ≈ queen
  - 相似含义的词在向量空间中距离更近
